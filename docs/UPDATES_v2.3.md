# v2.3 更新说明

## 主要变化

### 🏗️ 硬件架构细化

#### 背景
TPU 和昇腾都已发展多代，统一表示不够精确。

#### 改进

**Google TPU 扩展** (1 → 4)
```
之前: TPU (v3/v4/v5)

现在:
├─ TPU v3 (2018)
├─ TPU v4 (2020)
├─ TPU v5e (2023) - 效率版
└─ TPU v5p (2023) - 性能版
```

**华为昇腾扩展** (1 → 3)
```
之前: 昇腾 (910/910B)

现在:
├─ 昇腾 910 (2019)
├─ 昇腾 910B (2022)
└─ 昇腾 910C (2024)
```

#### 数据完善

**不同代际的支持差异**:
- TPU v5p 对 FP8 有原生支持，v3/v4 无支持
- 昇腾 910B/910C 对 BF16 有原生支持，910 仅部分支持
- 昇腾 910C 对 FP8 有部分支持
- TPU v4/v5 对 INT4 有更好的支持

#### 矩阵扩展
```
硬件列数: 10 列 → 16 列

新增列:
- Google: TPU v3, v4, v5e, v5p (原来是1个TPU)
- 华为: 昇腾 910, 910B, 910C (原来是1个昇腾)
```

### 🔍 交互简化

#### 删除所有筛选功能

**原因**:
1. 格式数量少（16个），完全可以一屏浏览
2. 卡片本身已有清晰的分类和用途标签
3. 筛选器增加操作成本，实际价值有限
4. 追求极致简洁，零学习成本

**之前 (v2.2)**:
```
格式详情页面:
[分类▼] [用途▼] [厂商▼]
─────────────────────────
[格式卡片网格...]
```

**现在 (v2.3)**:
```
格式详情页面:
格式详情
─────────────────────────
[格式卡片网格...]
```

#### 用户如何查找？

**方式 1: 直接浏览**
- 16 个卡片一屏可见
- 卡片上有分类标签和用途标签
- 视觉扫描速度 > 操作筛选器

**方式 2: 浏览器搜索**
- Ctrl/Cmd + F 使用浏览器内置搜索
- 功能更强大
- 用户更熟悉

## 使用场景更新

### 场景: 查找支持 BF16 的 Google 硬件

**v2.2**:
```
硬件矩阵 → TPU 列 → BF16 行 → 看到"原生支持"
```

**v2.3**:
```
硬件矩阵 → 看到 4 个 TPU 列
├─ TPU v3 → 原生支持
├─ TPU v4 → 原生支持

**v2.3**:
```

- 更好的升级决策
