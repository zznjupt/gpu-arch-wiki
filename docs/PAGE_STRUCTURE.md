# 页面结构说明

## 📄 单页面设计

整个网站只有**一个页面**，所有内容通过滚动浏览。

## 🗺️ 内容顺序

```
数据精度格式知识图谱
│
└─ 📊 格式总览（单页面）
    │
    ├─ 1️⃣ 统计数据
    │   ├─ 16 种精度格式
    │   ├─ 5 大厂商
    │   └─ 16 个硬件系列
    │
    ├─ 2️⃣ 核心概念（8个卡片）
    │   ├─ 位宽 (Bit Width)
    │   ├─ 精度 (Precision)
    │   ├─ 动态范围 (Dynamic Range)
    │   ├─ 混合精度 (Mixed Precision)
    │   ├─ 指数位 (Exponent)
    │   ├─ 尾数位 (Mantissa)
    │   ├─ 量化 (Quantization)
    │   └─ IEEE 754
    │
    ├─ 3️⃣ 格式详情（16个卡片）← 核心内容
    │   │
    │   ├─ 标准浮点 (3个)
    │   │   ├─ FP64
    │   │   ├─ FP32
    │   │   └─ FP16
    │   │
    │   ├─ AI 专用 (4个)
    │   │   ├─ TF32
    │   │   ├─ BF16
    │   │   ├─ FP8 E4M3
    │   │   └─ FP8 E5M2
    │   │
    │   ├─ 量化格式 (7个)
    │   │   ├─ INT16
    │   │   ├─ INT8
    │   │   ├─ INT4
    │   │   ├─ INT1
    │   │   ├─ FP4
    │   │   ├─ NF4
    │   │   └─ MXFP4
    │   │
    │   └─ 扩展格式 (2个)
    │       ├─ Posit
    │       └─ Block FP
    │
    └─ 4️⃣ 硬件支持（矩阵）
        ├─ 颜色图例
        └─ 支持矩阵（16列 × 9行）
```

## 🎯 浏览流程

### 推荐浏览顺序

```
1. 查看统计数据（了解整体）
   ↓
2. 阅读核心概念（理解基础）
   ↓
3. 浏览格式卡片（核心内容）
   ├─ 查看位结构预览
   ├─ 识别分类和用途标签
   └─ 点击感兴趣的格式
   ↓
4. 模态框详细信息
   ├─ 位结构详细图
   ├─ 性能数据
   ├─ 优缺点
   └─ 代码示例
   ↓
5. 向下滚动查看硬件支持（可选）
   └─ 查看各格式的硬件兼容性
```

### 快速查找流程

**场景 1: 我想了解 BF16**
```
向下滚动到"格式详情" → 找到 BF16 卡片 → 点击
→ 查看位结构、代码示例
```

**场景 2: 我想知道 H100 支持什么**
```
向下滚动到"硬件支持" → 找到 Hopper 列
→ 查看绿色圆点（原生支持的格式）
```

**场景 3: 我想找适合训练的格式**
```
向下滚动到"格式详情" → 查看卡片上的"训练"标签
→ FP64, FP32, TF32, BF16, FP16, FP8 都有
```

## 🎨 视觉层次

### 优先级从高到低

1. **统计数据** - 页面顶部，快速了解
2. **核心概念** - 基础知识，必读
3. **格式详情** - 核心内容，重点 ⭐
4. **硬件支持** - 辅助参考，可选

### 视觉强调

- **格式详情**: 卡片网格，显眼的位结构预览
- **硬件支持**: 紧凑的表格，不抢夺注意力
- **核心概念**: 统一的卡片样式，清晰易读

## 💡 设计理念

### Why 格式详情在前？

1. **用户最关心** - 来看页面的用户主要想了解各种格式
2. **视觉吸引** - 彩色的位结构预览吸引点击
3. **交互核心** - 点击卡片是主要交互
4. **内容丰富** - 16个卡片 + 模态框，信息量最大

### Why 硬件支持在后？

1. **辅助性质** - 硬件支持是选型的参考，不是主要内容
2. **表格密集** - 矩阵信息密度高，不适合放前面
3. **使用频率** - 只有选型时才需要查看
4. **视觉平衡** - 避免页面顶部过于拥挤

### Why 单页面？

1. **内容不多** - 16个格式，一屏可见
2. **逻辑连贯** - 概念 → 格式 → 硬件，顺序阅读
3. **操作简单** - 无需思考去哪个页面
4. **加载更快** - 一次加载所有内容

## 📱 响应式适配

### 桌面端 (> 768px)
- 格式卡片: 3-4 列网格
- 核心概念: 4 列网格
- 硬件矩阵: 横向滚动（16列）

### 移动端 (< 768px)
- 格式卡片: 1 列
- 核心概念: 1-2 列
- 硬件矩阵: 横向滚动

估算：
- 统计数据: ~200px
- 核心概念: ~400px
