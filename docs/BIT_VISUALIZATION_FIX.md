# 位结构可视化优化说明

## 🔧 修复的问题

### 问题 1: 短格式文字重叠
FP8 等 8 位格式的位结构图中，标签文字相互重叠或显示不完整。

### 问题 2: 大格式显示不如原版
FP64/FP32/TF32 的显示效果不如之前版本清晰。

### 问题 3: 可变位结构显示不当
Posit 和 Block FP 的位结构是可变的，显示固定位图没有意义。

## ✅ 解决方案

### 1. 自适应位框宽度

为不同位宽的格式使用不同的布局策略：

| 格式类型 | 总位数 | 位框宽度 | 总宽度 | 效果 |
|---------|--------|----------|--------|------|
| FP64 | 64位 | 15px | 1000px | 紧凑清晰 |
| FP32 | 32位 | 24px | 800px | 标准布局 |
| FP16/BF16/TF32 | 16-19位 | 38px | 750px | 舒适 |
| FP8 | 8位 | 70px | 650px | 宽松 ✅ |

**策略**:
- 位数越多，位框越窄（避免过宽）
- 位数越少，位框越宽（避免重叠）
- 保持总体宽度相近（650-1000px）

### 2. 智能标签显示

```javascript
// 根据段的实际宽度决定标签格式
if (boxWidth * exponentBits > 80 || totalBits <= 16) {
    // 空间充足：显示完整标签
    "exponent (8 bits)"
} else {
    // 空间紧凑：显示简化标签
    "exponent (8)"
}
```

### 3. 可变位结构特殊处理

**Posit 和 Block FP**:
- 模态框：不显示位图
- 卡片预览：显示"位结构可变"说明（虚线边框）

```
┌─────────────────────┐
│ Posit     [扩展格式] │
├─────────────────────┤
│ ┌─────────────────┐ │
│ │  位结构可变     │ │ ← 虚线边框 + 斜体
│ └─────────────────┘ │
│ 可变位  E可变  M可变 │
└─────────────────────┘
```

### 4. 位值显示优化

根据位框宽度调整位值字体：
- `boxWidth >= 35px`: 16px 字体
- `boxWidth >= 20px`: 13px 字体
- `boxWidth < 20px`: 11px 字体或不显示

### 5. SVG 居中显示

```css
.bit-visualization {
    display: flex;
    justify-content: center;
    align-items: center;
    min-height: 160px;
}

.bit-visualization svg {
    max-width: 100%;
    height: auto;
}
```

## 🎨 视觉改进

### 居中对齐
所有位图在模态框中居中显示，视觉更统一

### 间距优化
- 标签到位框: 15px
- 位框到位索引: 15px
- 位索引到底部标签: 25px

### 文字清晰度
- 标签字体: 12-13px（根据格式）
- 位值字体: 13-18px（根据位框大小）
- 位索引字体: 11px

## 💻 技术细节

### 布局计算

```javascript
// FP64 (64位)
boxWidth = 15px
totalWidth = 64 × 15 = 960px (实际 1000px)

// FP32 (32位)
boxWidth = 24px
totalWidth = 32 × 24 = 768px (实际 800px)

// FP8 (8位)
boxWidth = 70px
totalWidth = 8 × 70 = 560px (实际 650px)
```

### 响应式处理

SVG 使用 `viewBox` 和 `max-width: 100%`：
- 桌面端: 显示完整宽度
- 移动端: 自动缩放适配
