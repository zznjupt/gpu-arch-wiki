# 快速开始

## 📂 打开方式

```bash
# 方式 1: 直接双击
cd web && open index.html

# 方式 2: 本地服务器
cd web && python3 -m http.server 8000
# 访问 http://localhost:8000
```

## 🎯 3 分钟快速上手

### Step 1: 了解基础 (1.5分钟)
1. 打开页面，默认在"格式总览"
2. 阅读 8 个核心概念卡片
3. 向下滚动，查看硬件支持矩阵
4. 注意矩阵上方的颜色图例（🟢原生/完整/🟠部分/⚫不支持）

### Step 2: 查看格式 (1.5分钟)
1. 点击侧边栏"格式详情"
2. 浏览 16 种格式卡片（或使用筛选器）
3. 点击感兴趣的格式（如 BF16）
4. 查看模态框中的 6 项核心参数和代码示例

```

### 场景 5: BF16 的代码怎么写？

路径：

```

## 💡 实用技巧

### 筛选器组合
```
训练 + NVIDIA → FP32, TF32, BF16, FP16, FP8
量化 + 4位 → INT4, FP4, NF4, MXFP4
推理 + AMD → FP32, FP16, BF16, INT8
```

## ❓ FAQ

**Q: 图例中的"原生支持"是什么意思？**
A: 表示有专门的硬件加速单元（如 Tensor Core），性能最佳。

**Q: 为什么有些格式没有代码示例？**
A: 次要格式或实验性格式暂未添加，主要格式都已包含。

**Q: 相对性能数据准确吗？**
A: 这是近似值，实际性能取决于硬件、模型、框架等多种因素。仅供参考。

**Q: 可以离线使用吗？**
A: 可以，100% 纯静态，无任何外部依赖，完全离线可用。

**Q: 支持移动端吗？**
A: 支持，响应式设计，自动适配手机和平板。
