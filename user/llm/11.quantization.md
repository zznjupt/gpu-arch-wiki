### Quantization

`Quantization` 是把模型里的高精度数值换成更低精度表示。比如把 FP16/BF16 权重换成 INT8、INT4 或 FP8。

这样做有两个直接收益：

- 模型参数占用更少显存。
- 推理时从 HBM 读取的数据更少，带宽压力降低。

对大模型推理来说，很多时候瓶颈不是“乘加算不动”，而是“权重和 KV Cache 读得太慢”。低精度能减少这部分数据搬运。

### Scale

#### 低精度需要尺度信息

低精度整数本身不能直接表示原来的浮点范围，所以通常需要 scale：

$$
x \approx s \cdot q
$$

其中 `x` 是原始浮点值，`q` 是低精度整数，`s` 是缩放因子。

常见做法包括：

- per-tensor scale：整个张量共用一个 scale，实现简单。
- per-channel scale：每个输出通道有自己的 scale，精度更好。
- per-group scale：一组元素共用一个 scale，在精度和开销之间折中。

scale 越细，量化误差越小，但额外元数据和 kernel 实现也会更复杂。

### Weights

#### 权重量化最常见

权重量化通常最容易落地，因为模型权重是固定的，可以提前离线处理：

```text
FP16 weights
  -> calibration / quantization
  -> INT8 or INT4 weights
  -> inference kernel
```

权重量化能显著降低模型加载和矩阵乘读取权重的成本。对于 decode 阶段，这尤其重要，因为每生成一个 token 都要经过整套模型层。

### Activations

#### 激活量化更敏感

Activation 是推理过程中动态产生的，中间值分布会随输入变化。LLM 中还常有 outlier 激活值，直接用低精度表示可能导致误差变大。

因此 activation quantization 往往需要更谨慎的 calibration、混合精度策略，或者专门处理异常通道。

```text
权重: 固定，适合离线量化
激活: 动态，分布更难控制
KV Cache: 长序列下很大，也可以考虑低精度存储
```

### Tradeoff

Quantization 的核心取舍是：

- 精度越低，显存和带宽越省。
- 精度越低，量化误差越大，对模型质量越敏感。
- 硬件和 kernel 是否支持对应数据类型，会直接决定实际速度收益。
- 不同层、不同张量对低精度的敏感度不同，常常需要混合精度。

### Summary

Quantization 通过低精度表示减少模型权重、激活和缓存的数据量。它是 LLM 推理最常见的压缩和加速手段之一，但真正有效的量化不只是改数据类型，还要处理 scale、误差、硬件 kernel 和模型质量之间的平衡。
