### Speculative Inference

`Speculative Inference` 的思路是：用一个更小、更快的 draft model 先猜一串 token，再让原来的大模型一次性验证这些 token。

如果大模型同意 draft model 猜的前几个 token，就可以一次接受多个 token；如果中途不同意，就在第一个不匹配的位置停下来，丢掉后面的猜测。

```text
draft model:  猜 t1, t2, t3, t4
large model:  一次验证这几个候选 token
结果:
  t1, t2 接受
  t3 不接受
  t4 丢弃
```

它要加速的是自回归 decode 的串行性：普通 decode 一次只能确定一个 token，speculative inference 希望一次推进多个 token。

### Draft

#### 小模型先快速试探

Draft model 必须比目标大模型快很多，否则多跑一个模型只会增加开销。它可以是一个更小的模型，也可以是同一模型里更轻的预测头。

Draft 阶段会连续生成多个候选 token：

```text
当前上下文
  -> draft token 1
  -> draft token 2
  -> draft token 3
  -> draft token 4
```

这些 token 只是猜测，还没有被最终接受。

### Verify

#### 大模型并行验证候选 token

大模型验证时不需要像普通 decode 那样一个 token 一个 token 地完整跑多次。给定 draft 生成的候选序列后，大模型可以在一次 forward 里计算这些位置的预测分布。

```text
上下文 + [draft tokens]
  -> large model forward
  -> 检查每个 draft token 是否能被接受
```

只要前面的候选 token 被接受，后面的验证结果才有意义。遇到第一个不接受的位置后，从那里开始重新生成。

### Speedup

#### 什么时候会变快

Speculative inference 的收益取决于两个条件：

- draft model 足够快。
- draft tokens 被大模型接受的比例足够高。

如果每次 draft 的 4 个 token 里平均能接受 3 个，那么大模型一次 forward 就能推进多个 token，decode 延迟会下降。

如果 draft 经常猜错，大模型还是只能接受很短前缀，额外 draft 开销就不划算。

### Correctness

#### 目标不是换成小模型输出

Speculative inference 不是直接用小模型替代大模型。小模型只负责提出候选，大模型仍然负责最终验证。

在严格的 speculative sampling 设计里，通过接受/拒绝规则可以保持和目标大模型一致的采样分布。工程上也可以用更简单的贪心验证版本，但核心思想不变：最终输出必须经过大模型确认。

### Summary

Speculative Inference 用快模型先猜、多 token 提案，再用大模型批量验证。它把 decode 的一部分串行步骤变成“猜测 + 验证”，在 draft 够快且命中率高时可以显著降低延迟。
