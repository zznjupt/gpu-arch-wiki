### Distillation

`Distillation` 是用一个大模型教一个小模型。大模型通常叫 `teacher model`，小模型叫 `student model`。

目标不是把 teacher 的参数压缩成一个文件，而是让 student 在训练时学习 teacher 的行为。训练完成后，真正部署的是更小、更快的 student。

```text
teacher model: 大、慢、能力强
        ↓ 产生软标签/输出分布/中间信号
student model: 小、快、部署成本低
```

### Teacher

#### 学的不只是正确答案

普通监督训练只告诉模型最终答案是什么。Distillation 还会让 student 学 teacher 的输出分布。

比如分类或 next-token prediction 里，teacher 不只是说“正确 token 是 A”，还会给出很多候选 token 的概率：

```text
token A: 0.62
token B: 0.20
token C: 0.08
...
```

这些概率里包含了 teacher 对相似答案、次优答案和不确定性的判断。student 学到的不只是标签，而是 teacher 的偏好结构。

### Signals

#### 可以蒸馏哪些信息

常见 distillation 信号包括：

- logits：让 student 的输出分布接近 teacher。
- hidden states：让中间层表示接近 teacher。
- attention maps：让注意力分布接近 teacher。
- generated data：用 teacher 生成训练样本，再训练 student。

LLM 场景里，经常会把 teacher 生成的高质量问答、推理过程或偏好数据拿来训练更小的模型。

### Inference

#### 它为什么能优化推理

Distillation 的推理收益来自 student 更小：

- 参数更少，显存占用更低。
- 层数或 hidden dimension 更小，单 token 计算更快。
- KV Cache 也可能更小，因为层数、head 数或 hidden dimension 下降。

它和 quantization、sparsity 不同：distillation 改的是模型本身，而不是同一个模型的存储格式或稀疏模式。

### Tradeoff

Distillation 的核心取舍是：

- student 越小，推理越便宜。
- student 太小，会丢掉 teacher 的能力。
- 训练数据、蒸馏目标和 teacher 质量会直接影响 student 上限。
- 它需要重新训练，不是部署时换一个 kernel 就能完成。

### Summary

Distillation 用大模型训练小模型，以模型能力换取更低的推理成本。它适合需要长期部署、成本敏感、延迟敏感的场景，但需要额外训练流程来尽量保留 teacher 的能力。
