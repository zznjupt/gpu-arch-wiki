### Grouped-Query Attention

`Grouped-Query Attention` 简写为 `GQA`。它位于标准 MHA 和 MQA 之间：不是每个 query head 都有自己的 `K/V`，也不是所有 query heads 共享同一组 `K/V`，而是几个 query heads 组成一组，共享一组 `K/V`。

[full-image: GQA.png]
[caption: Multi-Head、Grouped-Query、Multi-Query 的对比：GQA 让多个 query heads 共享更少的 K/V heads。]

```text
MHA: Q heads = 16, KV heads = 16
GQA: Q heads = 16, KV heads = 4
MQA: Q heads = 16, KV heads = 1
```

它的目标是保留更多 attention 表达能力，同时减少 KV Cache 和 decode 带宽。

### Groups

#### 多个 Q 共享一组 K/V

假设有 `H_q` 个 query heads，`H_{kv}` 个 KV heads，并且 `H_q` 可以被 `H_{kv}` 整除。每个 KV head 会服务一组 query heads：

$$
g = \frac{H_q}{H_{kv}}
$$

其中 `g` 是每组里的 query head 数。

如果第 `h` 个 query head 属于第 `r(h)` 个 KV group，那么：

$$
O_h =
\operatorname{softmax}
\left(
\frac{Q_hK_{r(h)}^T}{\sqrt{d_h}}
\right)V_{r(h)}
$$

当 `H_{kv} = H_q` 时，GQA 退化为 MHA；当 `H_{kv} = 1` 时，GQA 退化为 MQA。

### Memory

#### KV Cache 按 KV heads 缩放

GQA 的 KV Cache 大小主要跟 `H_{kv}` 有关：

$$
M_{KV}^{GQA}
\approx
\frac{H_{kv}}{H_q}
M_{KV}^{MHA}
$$

比如从 32 个 query heads 改成 8 个 KV heads，KV Cache 和 decode 读取量大约可以降到标准 MHA 的四分之一。

```text
32 Q heads, 32 KV heads -> MHA
32 Q heads,  8 KV heads -> GQA
32 Q heads,  1 KV head  -> MQA
```

### Quality

#### 为什么常用 GQA 而不是直接 MQA

MQA 最省 KV Cache，但共享太激进。GQA 给模型保留了多组 `K/V` 表示，通常比 MQA 更接近 MHA 的质量，同时仍然明显降低推理成本。

很多现代 LLM 会在架构设计阶段直接采用 GQA。这样模型从训练开始就学习“多个 query heads 共享一组 K/V”的表示方式，效果比推理时临时改结构更可靠。

### Summary

Grouped-Query Attention 是 MHA 和 MQA 的折中：多个 query heads 共享一组 `K/V`。它用较小的质量代价换来显著的 KV Cache 和带宽节省，因此成为长上下文推理里很常见的 attention 设计。
