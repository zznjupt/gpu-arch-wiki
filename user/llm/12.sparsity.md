### Sparsity

`Sparsity` 是让模型里的很多数值变成 0，然后跳过这些 0 的存储和计算。

如果一个矩阵里大量元素都是 0，理论上矩阵乘可以只处理非零元素：

```text
dense:
[1.2  0.7 -0.4  2.1]
[0.3 -1.0  0.8  0.5]

sparse:
[1.2  0   -0.4  0  ]
[0   -1.0  0    0.5]
```

但稀疏不等于自动更快。GPU 擅长规则、连续、密集的计算；如果稀疏模式太随机，索引开销和不规则访存可能抵消掉节省的乘加。

### Pruning

#### 怎么得到稀疏模型

稀疏通常来自 `pruning`：把不重要的权重剪掉，让它们变成 0。

```text
训练好的 dense model
  -> 判断哪些权重不重要
  -> 置零
  -> 微调恢复质量
  -> sparse model
```

判断“不重要”的方式可以很简单，比如按权重绝对值大小剪；也可以更复杂，比如结合梯度、激活或结构约束。

### Structured

#### 结构化稀疏更容易加速

推理加速更偏好 `structured sparsity`。它不是随便把任意位置变成 0，而是规定稀疏模式，让硬件和 kernel 可以高效执行。

一个典型例子是 `2:4 sparsity`：每 4 个连续权重里保留 2 个非零值，其余 2 个为 0。

```text
[a b c d] -> 只保留其中 2 个非零
```

这种模式比完全随机稀疏更规则，硬件可以更容易跳过 0，同时减少索引和调度开销。

### Memory

#### 稀疏也需要元数据

稀疏矩阵不能只存非零值，还需要知道这些非零值的位置：

```text
values  +  indices
```

结构化稀疏的好处之一是元数据更少，因为位置模式受约束。非结构化稀疏更灵活，但需要更多索引信息，访存也更不规则。

### Tradeoff

Sparsity 的核心取舍是：

- 稀疏度越高，理论计算量越少。
- 稀疏模式越规则，越容易被 GPU 高效利用。
- 剪枝会影响模型质量，通常需要微调恢复。
- 没有匹配的 kernel 或硬件支持时，稀疏模型不一定比 dense 模型快。

### Summary

Sparsity 通过把部分权重变成 0 来减少计算和存储。它的收益依赖稀疏模式、硬件支持和模型质量恢复。对推理来说，结构化稀疏通常比随机稀疏更现实。
