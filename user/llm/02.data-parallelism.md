### Data Parallelism

`Data Parallelism` 是把输入数据分到多张 GPU 上，每张 GPU 都保存一份完整模型副本，只处理自己那一份 batch。

```text
GPU0 <- batch 0
GPU1 <- batch 1
GPU2 <- batch 2
GPU3 <- batch 3
```

它最常见于训练，也经常在很多并行方案的上下文里被一起提到。和 pipeline parallelism、tensor parallelism 不同，它不切模型参数，而是切输入数据。

### How

#### 复制模型，分发 batch

每张 GPU 都有完整权重，前向和反向都独立执行自己的样本。训练结束后，再通过梯度同步把各 GPU 的更新合并。

```text
same weights on each GPU
  -> different data shards
  -> gradient synchronization
```

所以 data parallelism 很适合大 batch 训练，也很容易扩展到多卡。但它要求每张卡都能装下完整模型。

### Inference

#### 为什么推理里没那么核心

文章里提到它，是为了说明多 GPU 方案不止一种。对推理来说，data parallelism 当然也能用：不同请求可以分发到不同 GPU 上独立处理。

但它解决的是“横向分流请求”，不是“单个模型太大放不下”的问题。模型如果本身已经超出单卡显存，还是要依赖 pipeline/tensor/sequence 这类模型并行。

### Summary

Data Parallelism 是按数据切 batch，不按模型切参数。它更像是一种请求分发和训练扩展方式，在推理里能帮助扩吞吐，但不是单卡放不下大模型时的核心解法。
