### Multi-Head Attention

`Multi-Head Attention` 是 Transformer 里的标准 attention 形式。它不是只用一组 `Q/K/V`，而是把 hidden dimension 切成多个 head，每个 head 独立做 attention，最后再把结果拼起来。

[full-image: MHA.png]
[caption: 从 Scaled Dot-Product Attention 到 Multi-Head Attention：先做单头注意力，再把多个 head 的结果合并。]

```text
hidden state
  -> head 0: Q0 K0 V0 -> O0
  -> head 1: Q1 K1 V1 -> O1
  -> head 2: Q2 K2 V2 -> O2
  -> head 3: Q3 K3 V3 -> O3
  -> concat -> output projection
```

不同 head 可以关注不同模式：有的更关注近邻 token，有的更关注语法结构，有的更关注长距离依赖。

### Formula

#### 单个 head

对第 `h` 个 head：

$$
Q_h = XW_h^Q,\qquad
K_h = XW_h^K,\qquad
V_h = XW_h^V
$$

attention 输出为：

$$
O_h =
\operatorname{softmax}
\left(
\frac{Q_hK_h^T}{\sqrt{d_h}}
\right)V_h
$$

其中 `d_h` 是单个 head 的维度。

### Concat

#### 多个 head 合并

所有 head 的输出会沿 hidden dimension 拼接，然后再经过一个输出投影：

$$
O =
\operatorname{Concat}(O_1,\ O_2,\ \dots,\ O_H)W^O
$$

这里 `H` 是 query heads 的数量。标准 MHA 里，每个 query head 都有自己对应的 key head 和 value head：

```text
Q heads: H
K heads: H
V heads: H
```

这带来了表达能力，也带来了 KV Cache 压力。

### KV Cache

#### 每个 head 都要存 K/V

在 decode 阶段，历史 token 的 `K/V` 会被存进 KV Cache。标准 MHA 中，`K/V` head 数和 query head 数相同，所以 KV Cache 大小会随 head 数一起增长：

$$
M_{KV}
\propto
L
\times
H_{kv}
\times
D_h
$$

其中 `L` 是 sequence length，`H_{kv}` 是 KV heads 数量，`D_h` 是单个 head 的维度。

这就是 `Multi-Query Attention` 和 `Grouped-Query Attention` 要优化的地方：不一定每个 query head 都需要一套独立的 `K/V`。

### Summary

Multi-Head Attention 通过多个 head 提升表达能力，但标准形式会为每个 head 保存独立 K/V。长上下文和大 batch 下，KV Cache 会成为推理显存和带宽的核心压力。
